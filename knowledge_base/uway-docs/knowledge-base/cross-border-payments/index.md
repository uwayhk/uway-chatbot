# Cross-border Payments

**Compliance requirements for international payment services.**

## Overview

Cross-border payment services face complex regulatory requirements across multiple jurisdictions. This guide covers key compliance considerations for operating international payment rails.

## Key Regulatory Considerations

### 1. Licensing Requirements

**Hong Kong:**
- SVF License (Stored Value Facility) - HKMA
- MSO License (Money Service Operator) - Customs & Excise

**Singapore:**
- MPI (Major Payment Institution) - MAS
- SPI (Standard Payment Institution) - MAS

**EU:**
- EMI (Electronic Money Institution) License
- PI (Payment Institution) License

### 2. AML/CFT Obligations

- Cross-border transaction monitoring
- Correspondent banking due diligence
- Travel Rule compliance (FATF Recommendation 16)
- Sanctions screening (OFAC, UN, EU, HKMA)

### 3. Data Protection

- GDPR (EU data subjects)
- PDPO (Hong Kong)
- PDPA (Singapore)
- Cross-border data transfer mechanisms

### 4. FX Regulations

- Exchange control requirements
- Reporting thresholds
- Beneficial ownership disclosure

## Implementation Best Practices

1. **Multi-jurisdiction compliance mapping**
2. **Real-time sanctions screening**
3. **Transaction monitoring rules engine**
4. **Audit trail and reporting systems**
